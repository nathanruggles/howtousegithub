The Ultimate Guide to Sharing HTML Presentations

Go from a local file to a professional, shareable link with an email button. No coding knowledge required.

Let's Get Started!

This interactive guide will walk you through two main parts:

  1. Publishing Your Presentation Online: We'll use a free service called GitHub Pages to turn your local HTML file into a live, public website link.
  2. Creating a Clickable Email Button: We'll create a professional button for your Gmail signature that links directly to your live presentation.

Click the "Next Step" button to advance through the guide. You can't break anything, so let's begin!

Part 1, Step 1: Create a Free GitHub Account

GitHub is a platform that lets you store and share code. We'll use its free website hosting feature.

  1. Go to github.com/signup.
  2. Follow the on-screen instructions to create your free account.
  3. Once you are signed in and see the main dashboard, you are ready for the next step.

Part 1, Step 2: Create a New "Repository"

A repository (or "repo") is just a project folder for your files.

  1. On your GitHub dashboard, click the green "Create repository" button, or find the "+" icon in the top-right corner and select "New repository".
  2. Give your repository a short, memorable name (e.g., `client-report`).
  3. Ensure the repository is set to Public. This is required for the website to be free.
  4. Check the box that says "Add a README file".
  5. Click the final "Create repository" button.

Part 1, Step 3: Upload Your HTML File

Now, let's add your presentation file to the project.

  1. Inside your new repository, click the "Add file" button and choose "Upload files".
  2. Drag and drop your HTML presentation file (e.g., `ConnexionPointAnalysis.html`) onto the page.
  3. Scroll to the bottom and click the green "Commit changes" button. (This is just GitHub's way of saying "Save").

Part 1, Step 4: Activate GitHub Pages

This is the final step where you "turn on" the website feature.

  1. In your repository, click on the "Settings" tab.
  2. In the left sidebar, click on "Pages".
  3. Under "Branch", select `main` from the dropdown menu and leave the folder as `/root`.
  4. Click "Save".

Heads Up: After saving, the page may take 1-5 minutes to update. It will then show a green box at the top saying "Your site is live at..." with your public URL.

Part 1, Step 5: Get Your Correct URL

This is the most important step to get right. Your URL must include the full filename at the end.

The correct structure is:

https://{your-username}.github.io/{repository-name}/{your-filename.html}

Common Mistake & Fix

If your page is blank, it's almost always because the URL is missing the filename. The link GitHub gives you often stops at the repository name.

Incorrect: https://username.github.io/client-report/

Correct: https://username.github.io/client-report/ConnexionPointAnalysis.html

You now have a live link! The next step is an optional pro-tip to make this link even better.

Part 1, Step 6 (Optional): Create a Shorter URL

If you prefer a cleaner URL without the `.html` filename at the end, you can rename your file to `index.html`. Web servers automatically look for this file name.

  1. Go to the main page of your repository (the "Code" tab).
  2. Click on your HTML file (e.g., `ConnexionPointAnalysis.html`) in the file list.
  3. In the top-right corner of the file view, click the **pencil icon** (Edit this file).
  4. At the top, change the filename from whatever it is to exactly `index.html`.
  5. Scroll to the bottom and click the green "Commit changes" button.

Result

After waiting a minute for the update, your shorter URL will work automatically.

New Shorter URL: https://username.github.io/client-report/

Part 2, Step 1: Prepare Your Button Code

Now let's create the button that will link to your live site.

  1. Copy the HTML code below into a simple text editor (like Notepad or TextEdit).
  2. Replace the placeholder `href` URL with your actual live GitHub URL. (Use the short one if you completed the optional step!)
<a href="https://your-username.github.io/repo-name/" style="background-color: #4f46e5; color: #ffffff; padding: 12px 20px; text-decoration: none; border-radius: 6px; font-family: sans-serif; font-size: 16px; font-weight: bold; display: inline-block;"> View Your Interactive Report </a>

Keep this code handy for the next step.

Part 2, Step 2: Insert the Button in Gmail

Gmail doesn't have a simple "Insert HTML" button, so we'll use a clever trick with the browser's developer tools. It looks technical, but it's just a copy-and-paste job.

  1. In Gmail, compose a new email.
  2. In the email body, type a unique placeholder, like `INSERT_BUTTON_HERE`.
  3. Highlight your placeholder text with your mouse.
  4. Right-click on the highlighted text and select "Inspect" from the menu. A code panel will open.

Part 2, Step 3: Edit HTML and Paste

You're almost there! Just a few more clicks.

  1. In the "Inspect" panel, you will see a line of code highlighted (it will contain your placeholder text).
  2. Right-click on that specific highlighted line.
  3. From the menu, choose "Edit as HTML".
  4. A text box will appear. Delete everything inside it.
  5. Paste your button code (from the previous step) into this box.
  6. Click anywhere outside the code box or close the "Inspect" panel by clicking the `X`.

Success! Your professional, clickable button should now be visible in your email draft.
